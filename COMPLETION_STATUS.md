## 📊 Как работает система

### Архитектура
```
┌─ КЛИЕНТ (Browser) ─┐
│  auth-client.js    │──────┐
│  client.js         │      │ WebSocket
└────────────────────┘      │
                            ▼
┌─ СЕРВЕР (Node.js) ─────────────┐
│  websocket-server.js            │
│  ├─ handleAuthRequest()         │ (login/signup)
│  ├─ handleSavePlayerData()      │ (autosave)
│  └─ handleRestoreSession()      │ (reload)
└─────────────┬──────────────────┘
              │
              ▼
       ┌─ БД (JSON) ─┐
       │ accounts    │ (username + password)
       │ players     │ (coins, skills, etc)
       └─────────────┘
```

### Процессы

**Вход/Регистрация:**
1. Пользователь вводит username + password
2. Отправляется на сервер (WebSocket)
3. Сервер проверяет/создает аккаунт
4. Пароль хешируется и сохраняется
5. Возвращается accountId + игровые данные
6. Клиент сохраняет в localStorage
7. Игра начинается

**Автосохранение (каждые 30 сек):**
1. Игровой прогресс отправляется на сервер
2. Сервер обновляет database.json
3. Данные всегда актуальны

**Перезагрузка страницы (F5):**
1. Браузер восстанавливает сессию из localStorage
2. Отправляет restoreSession на сервер
3. Сервер загружает данные из БД
4. Игра продолжается с того же места

## 🔐 Безопасность

### Текущая реализация (разработка):
- Base64 хеширование пароля + salt

- Сессия хранится в localStorage
