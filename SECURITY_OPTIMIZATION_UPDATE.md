# 🚀 SECURITY & OPTIMIZATION UPDATE

## ✅ Выполненные улучшения

### 1. 🔐 bcrypt вместо Base64 (Безопасность паролей)

**Что изменено:**
- Заменён простой Base64 хеш на bcrypt с 10 раундами соления
- Добавлена функция `verifyPassword()` для безопасной проверки паролей
- Все пароли теперь хранятся в виде криптографических хешей

**Файлы:**
- `auth.js` - обновлены функции хеширования
- `package.json` - добавлена зависимость `bcrypt: ^5.1.1`

**Использование:**
```javascript
// Хеширование пароля
const hash = await hashPassword('myPassword123');

// Проверка пароля
const isValid = await verifyPassword('myPassword123', hash);
```

### 2. 🔒 Rate Limiting (Защита от DDoS)

**Что изменено:**
- Оптимизированы лимиты запросов для игровых действий
- Разделение лимитов по типам сообщений:
  - Игровые действия (click, updateScore): до 500 запросов/минуту
  - Обычные запросы: 100 запросов/минуту
  - Аутентификация: 10 попыток/15 минут
- WebSocket лимиты: 200 подключений/IP, 100 сообщений/минуту

**Файлы:**
- `middleware/rate-limiter.js` - умная система лимитирования
- `websocket-server.js` - интеграция с WebSocket

**Особенности:**
- Игровые действия не блокируются строго (важно для игры)
- Автоматическая очистка данных при отключении
- Поддержка разных лимитов для разных типов запросов

### 3. 🗄️ Database.json → PostgreSQL (Базы данных)

**Что изменено:**
- Добавлен адаптер для работы с PostgreSQL
- Автоматическое переключение между JSON и PostgreSQL
- Создание таблиц при старте сервера
- Индексы для оптимизации производительности

**Файлы:**
- `middleware/database-adapter.js` - PostgreSQL адаптер
- `render.yaml` - конфигурация PostgreSQL на Render
- `package.json` - добавлена зависимость `pg: ^8.11.3`

**Таблицы:**
- `accounts` - аккаунты пользователей
- `players` - игровые данные
- `events` - события и ивенты
- `event_coins` - игровые валюты ивентов

**Настройка:**
```bash

```

**Режимы работы:**
- Если `DATABASE_URL` не установлен → используется `database.json`
- Если `DATABASE_URL` установлен → используется PostgreSQL

### 5. 💾 Backup стратегия (Автоматические бэкапы)

**Что изменено:**
- Скрипт автоматического бэкапа базы данных
- Сжатие бэкапов (gzip)
- Ограничение количества бэкапов (по умолчанию 7)
- CLI команды для управления бэкапами

**Файлы:**
- `scripts/backup-db.js` - система бэкапов
- `package.json` - скрипт `npm run backup`

**Команды:**
```bash
# Создать бэкап
node scripts/backup-db.js create

# Восстановить из бэкапа
node scripts/backup-db.js restore backups/backup-2024-01-01.json.gz

# Показать список бэкапов
node scripts/backup-db.js list

# Автоматический бэкап каждые 24 часа
node scripts/backup-db.js auto 24
```

**Переменные окружения:**
```bash
BACKUP_DIR=./backups        # Директория для бэкапов
MAX_BACKUPS=7               # Максимум бэкапов
AUTO_BACKUP=true            # Включить авто-бэкап
BACKUP_INTERVAL_HOURS=24    # Интервал бэкапов
```

## ⚠️ Важные заметки

### Миграция с JSON на PostgreSQL:
1. Создайте PostgreSQL базу на Render
2. Настройте `DATABASE_URL`
3. Сервер автоматически создаст таблицы
4. Существующие данные нужно будет импортировать вручную

### Бэкапы:
- Хранятся в `backups/` директории
- Не деплоятся на сервер (в `.gitignore`)
- Рекомендуется настроить внешний storage (S3 и т.п.)

### Rate Limiting:
- Игровые действия имеют мягкие лимиты
- При нарушении лимита игрок получает предупреждение
- IP бан не применяется автоматически
