# 🎮 Админка для Игры

Веб-интерфейс для управления данными игры в PostgreSQL.

## 🚀 Запуск

### 1. Установите зависимости
```bash
npm install
```

### 2. Настройте переменные окружения
```bash
# .env
DATABASE_URL=postgresql://user:password@localhost:5432/yourdb
ADMIN_PASSWORD=your_secure_password_here
ADMIN_PORT=3002
```

### 3. Запустите серверы

**Запустить игру и админку вместе:**
```bash
npm run start-all
```

**Или отдельно:**
```bash
# Только игра
npm start

# Только админка
npm run admin
```

### 4. Откройте в браузере
```
Игра: http://localhost:3000
Админка: http://localhost:3002/admin.html
```

## 🔑 Авторизация

Введите пароль администратора (из переменной `ADMIN_PASSWORD` или `admin123` по умолчанию).

## ⚠️ Безопасность

1. **Смените пароль по умолчанию!**
```bash
ADMIN_PASSWORD=your_super_secure_password
```

2. **В production используйте HTTPS**
3. На Vercel пароль берётся из `ADMIN_PASSWORD` в настройках проекта

## 📋 Возможности

### Просмотр игроков
- Список всех игроков (до 100)
- Статистика: монеты, уровень, клики
- Поиск по имени или ID
- Фильтрация забаненных игроков

### Редактирование
- Монеты и totalCoins
- Уровень
- Монет за клик / в секунду
- Количество кликов

### Управление доступом
- Бан/разбан игроков
- Указание причины бана
- Удаление игроков

### Массовые операции
- Добавить монеты нескольким игрокам
- Установить уровень нескольким игрокам
- Забанить несколько игроков

| Method | Endpoint | Описание |
|--------|----------|----------|
| GET | `/api/admin/players` | Список игроков |
| GET | `/api/admin/players/:id` | Данные игрока |
| PUT | `/api/admin/players/:id` | Обновить игрока |
| DELETE | `/api/admin/players/:id` | Удалить игрока |
| POST | `/api/admin/ban/:id` | Забанить |
| POST | `/api/admin/unban/:id` | Разбанить |
| GET | `/api/admin/stats` | Статистика |

## 💡 Советы

- Регулярно делайте бэкапы БД перед массовыми правками
- Используйте поиск чтобы быстро найти игрока
- Массовые операции удобны для корректировки баланса
- Баньте с указанием причины для отладки
